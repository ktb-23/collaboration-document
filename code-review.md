## 팀 코드 리뷰 가이드라인

## 라벨 사용 규칙
코드 리뷰 시 각 PR에 다음과 같은 라벨을 적용합니다.

### 우선 순위
우선순위와 관련된 라벨은 다음과 같습니다.
- P-1: 오늘 내로 리뷰가 필요한 작업입니다. 최대한 빠르게 작업을 진행합니다.
- P-2: 기본 우선 순위입니다.
- P-3: 중요하지 않은 리뷰에대한 요청입니다. 최후순위로 리뷰를 진행합니다.

### 마감일
기본적으로 3일 이내에 리뷰를 진행합니다.

마감일과 관련된 라벨은 다음과 같습니다.

- OVERDUE: 마감일이 지난 경우 입니다.
- D-0:`P-1` 우선순위의 경우 적용됩니다. 최우선순위로 작업을 나타냅니다.
- D-n: `P-2` 우선순위를 가지는 경우 경과 시간에 따라 표기됩니다.

## 작성 방식

### 권장사항 표기
다음 내용은 [뱅크샐러드 개발 블로그](https://blog.banksalad.com/tech/banksalad-code-review-culture/#%EC%BB%A4%EB%AE%A4%EB%8B%88%EC%BC%80%EC%9D%B4%EC%85%98-%EB%B9%84%EC%9A%A9%EC%9D%84-%EC%A4%84%EC%9D%B4%EA%B8%B0-%EC%9C%84%ED%95%9C-pn-%EB%A3%B0)를 참조하였습니다

리뷰의 중요도나 권장 사항의 수준을 나타냅니다.

가장 높은 권장사항 수준에 맞게 Request changes, Comment, Approve를 결정합니다.

- **L-1: 꼭 반영해주세요 (Request changes)**
  - PR의 내용이 서비스에 중대한 오류를 발생시킬 수 있는 가능성이 있는 경우.
  - 리뷰어는 P1 태그를 통해 리뷰 요청자에게 수정을 요청합니다.
  - 리뷰 요청자는 P1 태그에 대해 리뷰어의 요청을 반영하거나, 반영할 수 없는 합리적인 이유를 통해 리뷰어를 설득해야 합니다.

- **L-2: 적극적으로 고려해주세요 (Request changes)**
  - PR 내용에 대해 개선이 필요할 때 사용.
  - 작성자는 P2에 대해 수용하거나, 수용할 수 없는 상황이라면 적합한 의견을 들어 토론할 것을 권장합니다.

- **L-3: 웬만하면 반영해 주세요 (Comment)**
  - 리뷰어가 제안하는 개선 사항이 있지만, 반드시 반영해야 하는 것은 아닌 경우.
  - 작성자는 P3에 대해 수용하거나, 반영할 수 없는 이유를 설명하거나 다음에 반영할 계획을 명시적으로 표현합니다. Request changes가 아닌 Comment와 함께 사용됩니다.

- **L-4: 반영해도 좋고 넘어가도 좋습니다 (Approve)**
  - 리뷰어의 의견이 꼭 반영될 필요는 없으며, 작성자가 고려할 가치가 있다고 생각할 때.
  - 작성자는 P4에 대해서는 의견을 달지 않고 무시해도 괜찮습니다.

- **L-5: 그냥 사소한 의견입니다 (Approve)**
  - 리뷰어의 의견이 사소하며, 반영 여부가 중요하지 않은 경우.
  - 작성자는 P5에 대해 아무런 의견을 달지 않고 무시해도 괜찮습니다.

## 리뷰어 선정

### 페어 개발자 필수 리뷰
- 동일한 파트에 있는 페어 개발자는 서로의 필수 리뷰어입니다.

### 추가 코드 리뷰 요청
- 도움이 필요한 경우 다른 파트의 개발자에게 코드 리뷰를 요청할 수 있습니다.
- 요청은 디스코드의 요청 채널을 사용합니다.

## 추가 사항

### 리뷰 시간 할애
- 매일 16:00 - 18:00을 리뷰 시간으로 설정합니다.
- 추가된 리뷰는 해당 시간에 꼭 확인합니다.
- 이를 통해 `P-0` 수준의 급한 작업을 당일 처리할 수 있습니다.
- 해당 규칙은 품질 향상을 위한 중요한 문화입니다.

### 주석 작성 및 설명
- PR 요청 시 리뷰어를 위해 최대한 자세히 설명을 작성합니다.
- 이해하기 어려운 코드는 미리 주석을 달아둡니다.

### 피리뷰어 배려
- 피리뷰어의 입장을 고려하여 리뷰어는 배려하는 말투를 사용합니다.
- 피리뷰어의 피로도를 줄이기 위해 리뷰의 최종 정리에만 칭찬을 허용합니다.
- 나머지 부분은 토론을 위한 질문 혹은 수정 요청으로 제한합니다.

### 작은 단위로 코드 리뷰 요청
- 코드 리뷰 요청은 작은 단위로 나눠서 요청합니다.
